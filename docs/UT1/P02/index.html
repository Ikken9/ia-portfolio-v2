Feature Engineering

3 min read

La práctica consiste en construir y evaluar un modelo de clasificación predictivo (clasificador) utilizando un dataset real, siguiendo las etapas iniciales de un pipeline de Machine Learning.

Objetivo

  • Crear un modelo Logistic Regression y Baseline
  • Comparar los modelos
  • Analizar los resultados obtenidos

Modelo base y baseline

Además del modelo de Logistic Regression es necesaria la creación de un modelo baseline para poder compararlos y evaluar el desempeño del modelo más complejo (Logistic Regression). Si el modelo sofisticado no puede superar al baseline, entonces muy probablemente algo esté mal.

Preguntas

¿En qué casos se equivoca más el modelo: cuando predice que una persona sobrevivió y no lo hizo, o al revés?

Basado en la matriz de confusión, el modelo se equivoca más prediciendo supervivencia cuando en realidad la persona murió (falsos positivos) que al revés (falsos negativos).

¿El modelo acierta más con los que sobrevivieron o con los que no sobrevivieron?

El modelo acierta más con las personas que no sobrevivieron (clase mayoritaria). Detecta mejor a quienes fallecieron que a quienes sobrevivieron.

¿La Regresión Logística obtiene más aciertos que el modelo que siempre predice la clase más común?

El modelo acierta más con los que no sobrevivieron que con los que sobrevivieron.

  • No supervivientes (clase 0) recall = 0.89
  • Supervivientes sobrevivieron (clase 1) recall = 0.70

¿Cuál de los dos tipos de error creés que es más grave para este problema?

Bajo mi punto de vista, el error más grave en este problema es un falso negativo: predecir que una persona no sobrevivió cuando en realidad sí lo hizo. Esto sería crítico si el modelo se usara, por ejemplo, para simulaciones de seguridad.

Mirando las gráficas y números, ¿qué patrones interesantes encontraste sobre la supervivencia?

  • La clase mayoritaria (los que no sobrevivieron) se predice mejor.
  • Las mujeres y pasajeros de primera clase suelen estar más representados entre los sobrevivientes.
  • El modelo logra buen balance entre precisión y recall, pero falla bastante en sobrevivientes.

¿Qué nueva columna (feature) se te ocurre que podría ayudar a que el modelo acierte más?

Estaría buenísimo si el dataset tuviese una columna "IsAlone" para saber si la persona viajaba sola o acompañada. También sería interesante el tener una feature con el título extraído del nombre, por ej.: "Mr.", "Miss", etc.
